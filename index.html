1. 基礎概念

時間域 vs 頻率域 — 傅立葉分析的核心思想

1.1 時間域 (Time Domain) vs 頻率域 (Frequency Domain)

在工程分析中，我們觀察訊號有兩個基本視角：

  • 時間域 (Time Domain)：觀察訊號的振幅如何隨時間變化。示波器（Oscilloscope）顯示的就是時間域波形。
  • 頻率域 (Frequency Domain)：觀察訊號包含哪些頻率成分以及各成分的強度。頻譜分析儀（Spectrum Analyzer）顯示的就是頻率域資訊。

傅立葉分析的核心思想

任何「合理」的週期訊號都可以分解為不同頻率的正弦波（Sinusoid）之疊加。

$$x(t) = \sum_{k=-\infty}^{\infty} c_k \, e^{j k \omega_0 t}$$

1.2 為什麼 EE 工程師必須學傅立葉分析？

濾波器設計 (Filter Design)

在頻率域中，濾波器就是一個乘法運算。低通、高通、帶通濾波器的設計全部建立在頻率域分析之上。

通訊系統 (Communication Systems)

調變（Modulation）、解調（Demodulation）、頻寬分配（Bandwidth Allocation）都需要頻率域的觀點。

電力系統 (Power Systems)

分析電源的諧波失真（THD）、電力品質監測，都依賴傅立葉分析來識別有害的諧波成分。

音訊與振動 (Audio & Vibration)

語音辨識、音訊壓縮（MP3）、機械故障診斷都需要分析訊號的頻率組成。

互動圖表：時間域 ↔ 頻率域

調整三個正弦波的頻率與振幅，觀察合成訊號在時間域和頻率域的變化。

2. 傅立葉級數 (Fourier Series)

週期訊號的頻率分解

2.1 三角形式 (Trigonometric Form)

對於週期為 $T$、基頻為 $\omega_0 = 2\pi/T$ 的週期訊號 $x(t)$：

$$x(t) = \frac{a_0}{2} + \sum_{n=1}^{\infty} \left[ a_n \cos(n\omega_0 t) + b_n \sin(n\omega_0 t) \right]$$

其中：$a_n = \frac{2}{T}\int_0^T x(t)\cos(n\omega_0 t)\,dt$，$b_n = \frac{2}{T}\int_0^T x(t)\sin(n\omega_0 t)\,dt$

2.2 複數指數形式 (Complex Exponential Form)

$$x(t) = \sum_{n=-\infty}^{\infty} c_n \, e^{jn\omega_0 t}$$

其中：$c_n = \frac{1}{T}\int_0^T x(t)\,e^{-jn\omega_0 t}\,dt$

複數形式更加簡潔，在工程分析中使用更為廣泛。兩種形式之間的關係：$c_n = \frac{1}{2}(a_n - jb_n)$。

2.3 經典波形範例

以下是三種常見週期波形的傅立葉級數：

方波 (Square Wave)

$$x(t) = \frac{4}{\pi}\sum_{n=1,3,5,\ldots}^{\infty}\frac{1}{n}\sin(n\omega_0 t)$$

只包含奇數次諧波，振幅以 $1/n$ 衰減。

三角波 (Triangle Wave)

$$x(t) = \frac{8}{\pi^2}\sum_{n=1,3,5,\ldots}^{\infty}\frac{(-1)^{(n-1)/2}}{n^2}\sin(n\omega_0 t)$$

同樣只有奇數次諧波，但振幅以 $1/n^2$ 更快衰減（更平滑）。

鋸齒波 (Sawtooth Wave)

$$x(t) = \frac{2}{\pi}\sum_{n=1}^{\infty}\frac{(-1)^{n+1}}{n}\sin(n\omega_0 t)$$

包含所有次諧波（奇數與偶數），振幅以 $1/n$ 衰減。

互動實驗：諧波合成

選擇波形類型並調整諧波數量，觀察傅立葉級數的逐步逼近過程。注意在不連續點附近的 Gibbs 現象 (Gibbs Phenomenon)。

2.4 Gibbs 現象 (Gibbs Phenomenon)

當我們用有限項傅立葉級數逼近不連續的訊號時，在不連續點附近會出現約 9% 的過衝（overshoot）。即使諧波數量趨近無限大，這個過衝的百分比不會消失，只是寬度趨近於零。

工程啟示：Gibbs 現象解釋了為什麼數位訊號（方波）在實際電路中會產生 ringing — 因為任何實際系統都有有限頻寬，無法傳遞無限多的諧波。

Python / MATLAB 對應程式碼

import numpy as np
import matplotlib.pyplot as plt

t = np.linspace(0, 2, 1000)
N_harmonics = 20
x = np.zeros_like(t)

    x += (4/(np.pi*n)) * np.sin(2*np.pi*n*t)

plt.axhline(y=1, color='r', linestyle='--', alpha=0.5)
plt.title(f'方波傅立葉級數 (N={N_harmonics} 項)')
plt.xlabel('t'); plt.ylabel('x(t)')
plt.legend(); plt.grid(True); plt.show()

% 方波的傅立葉級數合成

end

yline(1, 'r--');
title(sprintf('方波傅立葉級數 (N=%d 項)', N_harmonics));
xlabel('t'); ylabel('x(t)'); grid on;

3. 連續時間傅立葉轉換 (CTFT)

Continuous-Time Fourier Transform — 非週期訊號的頻率分析

3.1 定義

將傅立葉級數推廣到非週期訊號，令週期 $T \to \infty$：

正轉換（Analysis）：

$$X(j\omega) = \int_{-\infty}^{\infty} x(t)\, e^{-j\omega t}\, dt$$

反轉換（Synthesis）：

$$x(t) = \frac{1}{2\pi}\int_{-\infty}^{\infty} X(j\omega)\, e^{j\omega t}\, d\omega$$

3.2 重要性質

性質時間域頻率域
線性 (Linearity)$ax_1(t)+bx_2(t)$$aX_1(j\omega)+bX_2(j\omega)$
時移 (Time Shift)$x(t-t_0)$$e^{-j\omega t_0}X(j\omega)$
頻移 (Frequency Shift)$e^{j\omega_0 t}x(t)$$X(j(\omega-\omega_0))$
尺度 (Scaling)$x(at)$$\frac{1}{|a|}X\!\left(\frac{j\omega}{a}\right)$
時域卷積 (Convolution)$x_1(t)*x_2(t)$$X_1(j\omega)\cdot X_2(j\omega)$
Parseval 定理$\int_{-\infty}^{\infty}|x(t)|^2\,dt = \frac{1}{2\pi}\int_{-\infty}^{\infty}|X(j\omega)|^2\,d\omega$

3.3 常見轉換對

訊號 $x(t)$轉換 $X(j\omega)$說明
$\text{rect}(t/\tau)$$\tau\,\text{sinc}(\omega\tau/2\pi)$矩形脈衝 ↔ sinc
$\text{sinc}(Wt/\pi)$$\frac{\pi}{W}\text{rect}(\omega/2W)$sinc ↔ 矩形（對偶性）
$\delta(t)$$1$脈衝函數 (Dirac Delta)
$1$$2\pi\delta(\omega)$常數 ↔ 頻域脈衝
$e^{-at}u(t),\; a>0$$\frac{1}{a+j\omega}$單邊指數衰減
$e^{-\alpha t^2}$$\sqrt{\frac{\pi}{\alpha}}\,e^{-\omega^2/(4\alpha)}$Gaussian ↔ Gaussian

互動圖表：矩形脈衝與 sinc 函數

調整矩形脈衝寬度 $\tau$，觀察時域變寬時頻域變窄（不確定性原理 Uncertainty Principle 的體現）。

4. 離散傅立葉轉換 (DFT) 與快速傅立葉轉換 (FFT)

Discrete Fourier Transform & Fast Fourier Transform

4.1 DFT 定義

對於長度為 $N$ 的離散序列 $x[n]$，其 DFT 定義為：

正轉換：

$$X[k] = \sum_{n=0}^{N-1} x[n]\, e^{-j2\pi kn/N}, \quad k = 0, 1, \ldots, N-1$$

反轉換（IDFT）：

$$x[n] = \frac{1}{N}\sum_{k=0}^{N-1} X[k]\, e^{j2\pi kn/N}, \quad n = 0, 1, \ldots, N-1$$

直接計算 DFT 的時間複雜度為 $O(N^2)$，因為需要 $N$ 個輸出，每個輸出需要 $N$ 次複數乘加。

4.2 FFT — 高效率的 DFT 演算法

快速傅立葉轉換 (Fast Fourier Transform, FFT) 並不是一種不同的轉換，而是計算 DFT 的高效演算法。最著名的是 Cooley-Tukey 演算法（1965 年）。

Radix-2 分解（$N$ 為 2 的冪次）

$$X[k] = \underbrace{\sum_{r=0}^{N/2-1} x[2r]\, W_N^{2rk}}_{\text{偶數項 DFT}} + W_N^k \underbrace{\sum_{r=0}^{N/2-1} x[2r+1]\, W_N^{2rk}}_{\text{奇數項 DFT}}$$

其中 $W_N = e^{-j2\pi/N}$ 為旋轉因子 (Twiddle Factor)

這個遞迴分解將時間複雜度從 $O(N^2)$ 降低到 $O(N\log_2 N)$。對於 $N = 1,\!048,\!576 = 2^{20}$，加速比為 $N/\log_2 N = 52,\!428$ 倍！

4.3 零填充 (Zero-Padding)

在原始訊號後面補零不會增加新的頻率資訊，但可以：

  • 增加頻率軸的取樣密度（更細的 frequency bin）→ 頻譜看起來更「平滑」
  • 讓 FFT 長度成為 2 的冪次 → 可使用 Radix-2 FFT
  • 等效於對 DTFT 做更密集的取樣

注意：零填充提高的是「頻率域的內插解析度」，而非真正的「頻率解析度」。真正的頻率解析度取決於訊號的實際觀測長度（視窗長度）。

互動實驗：FFT 與零填充效果

比較不同零填充倍數下的 FFT 頻譜。訊號：兩個正弦波（50 Hz + 55 Hz），取樣率 500 Hz，觀測長度 0.1 秒（50 個點）。

4.4 Radix-2 FFT JavaScript 實作

以下是本網站使用的 FFT 函式完整原始碼（已內建於頁面中）：

/**
 * Radix-2 Cooley-Tukey FFT
 * 輸入：real[] 實部陣列，imag[] 虛部陣列（長度必須為 2 的冪次）
 * 輸出：{ real: [], imag: [] } 頻域複數結果
 */
function fft(real, imag) {
  const N = real.length;
  if (N === 1) return { real: [...real], imag: [...imag] };

  // 位元反轉排序 (Bit-Reversal Permutation)
  const re = new Float64Array(N);
  const im = new Float64Array(N);
  const bits = Math.log2(N);
  for (let i = 0; i < N; i++) {
    let rev = 0;
    for (let b = 0; b < bits; b++) {
      rev = (rev << 1) | ((i >> b) & 1);
    }
    re[rev] = real[i];
    im[rev] = imag[i];
  }

  // 蝴蝶運算 (Butterfly Operations)
  for (let size = 2; size <= N; size *= 2) {
    const half = size / 2;
    const angle = -2 * Math.PI / size;
    for (let i = 0; i < N; i += size) {
      for (let j = 0; j < half; j++) {
        const wr = Math.cos(angle * j);  // 旋轉因子實部
        const wi = Math.sin(angle * j);  // 旋轉因子虛部
        const tr = wr * re[i+j+half] - wi * im[i+j+half];
        const ti = wr * im[i+j+half] + wi * re[i+j+half];
        re[i+j+half] = re[i+j] - tr;
        im[i+j+half] = im[i+j] - ti;
        re[i+j] += tr;
        im[i+j] += ti;
      }
    }
  }
  return { real: Array.from(re), imag: Array.from(im) };
}

5. 視窗函數 (Window Functions) 與頻譜洩漏 (Spectral Leakage)

這是頻譜分析中最關鍵的工程技巧之一

5.1 為什麼需要加窗？

在實際應用中，我們只能觀測有限長度的訊號。將無限長訊號截斷為有限長度，等同於將訊號乘以一個矩形窗 (Rectangular Window)。

$$x_w[n] = x[n] \cdot w[n]$$

時域相乘 ↔ 頻域卷積（Convolution）

$$X_w(e^{j\omega}) = X(e^{j\omega}) * W(e^{j\omega})$$

矩形窗的頻譜是 sinc 函數，具有很高的旁瓣（Sidelobes），這會導致頻譜洩漏 (Spectral Leakage) — 本來集中在一個頻率上的能量「洩漏」到鄰近頻率。

核心概念：使用其他形狀的視窗（如 Hann、Hamming、Blackman）可以減低旁瓣，從而減少頻譜洩漏，代價是主瓣（Main Lobe）變寬 → 頻率解析度降低。這就是「頻率解析度 vs 動態範圍」的根本取捨。

5.2 常見視窗函數公式

以下列出五種常見視窗的數學定義（$n = 0, 1, \ldots, N-1$）：

Rectangular（矩形窗）

$$w[n] = 1$$

Hann（漢寧窗）/ Hanning

$$w[n] = 0.5\left(1 - \cos\frac{2\pi n}{N-1}\right)$$

Hamming（漢明窗）

$$w[n] = 0.54 - 0.46\cos\frac{2\pi n}{N-1}$$

Blackman（布萊克曼窗）

$$w[n] = 0.42 - 0.5\cos\frac{2\pi n}{N-1} + 0.08\cos\frac{4\pi n}{N-1}$$

Kaiser（凱薩窗）

$$w[n] = \frac{I_0\!\left(\beta\sqrt{1-\left(\frac{2n}{N-1}-1\right)^2}\right)}{I_0(\beta)}$$

$I_0$ 為零階修正貝塞爾函數；$\beta$ 參數控制旁瓣衰減

5.4 視窗的時域與頻域波形

選擇視窗觀察其時域形狀和頻域響應（dB 尺度）。Kaiser 窗可調整 β 參數。

5.5 如何選擇視窗？工程指南

需要高頻率解析度

→ 使用窄主瓣視窗（Rectangular 或 Kaiser β 小）

場景：分辨兩個相鄰頻率、精確量測頻率值

需要高動態範圍

→ 使用低旁瓣視窗（Blackman 或 Kaiser β 大）

場景：偵測強訊號旁邊的微弱訊號、諧波失真分析

通用分析

→ Hann 窗是最安全的預設選擇

場景：一般頻譜分析、不確定需求時的首選

需要彈性調整

→ Kaiser 窗（調整 β 可在解析度與動態範圍間取捨）

場景：需要根據具體需求微調的專業分析

5.6 互動實驗：頻譜洩漏與分辨能力

兩個接近頻率的正弦波 + 可選雜訊。切換不同視窗，即時比較 FFT 頻譜的洩漏與頻率分辨能力。

Python / MATLAB 對應程式碼

import numpy as np
from scipy.signal import windows
import matplotlib.pyplot as plt

fs = 1000; N = 256
t = np.arange(N) / fs
f1, f2 = 100, 108
x = np.sin(2*np.pi*f1*t) + 0.5*np.sin(2*np.pi*f2*t)

# 比較不同視窗
for name, win in [('Rectangular', np.ones(N)),
                  ('Hann', windows.hann(N)),
                  ('Hamming', windows.hamming(N)),
                  ('Blackman', windows.blackman(N)),
                  ('Kaiser β=6', windows.kaiser(N, 6))]:
    xw = x * win
    X = np.fft.fft(xw, 4*N)
    freqs = np.fft.fftfreq(4*N, 1/fs)
    mag_dB = 20*np.log10(np.abs(X[:2*N]) / (N/2) + 1e-12)
    plt.plot(freqs[:2*N], mag_dB, label=name)

plt.xlim([50, 200]); plt.ylim([-80, 5])
plt.xlabel('Frequency (Hz)'); plt.ylabel('Magnitude (dB)')
plt.title('視窗函數比較'); plt.legend(); plt.grid(); plt.show()
fs = 1000; N = 256;
t = (0:N-1)'/fs;
f1 = 100; f2 = 108;
x = sin(2*pi*f1*t) + 0.5*sin(2*pi*f2*t);

wins = {ones(N,1), hann(N), hamming(N), blackman(N), kaiser(N,6)};
names = {'Rect','Hann','Hamming','Blackman','Kaiser'};

for i = 1:5
    xw = x .* wins{i};
    X = fft(xw, 4*N);
    freqs = (0:4*N-1)*(fs/(4*N));
    mag_dB = 20*log10(abs(X(1:2*N))/(N/2) + 1e-12);
    plot(freqs(1:2*N), mag_dB); hold on;
end
xlim([50 200]); ylim([-80 5]);
legend(names); xlabel('Hz'); ylabel('dB'); grid on;

6. 短時傅立葉轉換 (STFT) 與 Spectrogram

Short-Time Fourier Transform — 觀察訊號頻率隨時間的變化

6.1 STFT 定義

標準傅立葉轉換只能告訴我們訊號包含「哪些」頻率，但無法告訴我們「何時」出現。STFT 透過滑動視窗解決此問題：

$$\text{STFT}\{x[n]\}(m, \omega) = \sum_{n=-\infty}^{\infty} x[n]\, w[n-m]\, e^{-j\omega n}$$

$m$ 是時間索引（視窗中心位置），$w[n]$ 是視窗函數

6.2 重要參數

  • 視窗長度 (Window Length)：越長 → 頻率解析度越高，但時間解析度越低。越短 → 時間解析度高，但頻率解析度低。這是不確定性原理 (Uncertainty Principle) 的直接體現。
  • 重疊率 (Overlap)：相鄰視窗之間的重疊比例。常見值為 50%～75%。重疊率越高，時間軸取樣越密，但計算量越大。
  • FFT 長度 (NFFT)：通常 ≥ 視窗長度（多出的部分零填充），決定頻率軸的取樣密度。

不確定性原理：$\Delta t \cdot \Delta f \geq \frac{1}{4\pi}$。無法同時獲得完美的時間解析度和頻率解析度——必須根據應用需求做取捨。

6.3 工程應用

語音與音訊分析

語音辨識系統使用 Mel-Spectrogram 作為特徵輸入；音樂分析用 Spectrogram 觀察旋律與和聲。

振動分析與故障診斷

旋轉機械的軸承故障會在特定頻率產生週期性衝擊，Spectrogram 可追蹤這些特徵隨時間的變化。

互動實驗：Spectrogram

產生 Chirp 訊號（頻率隨時間線性增加）或雙頻訊號，觀察彩色 Spectrogram。

Python / MATLAB 對應程式碼

import numpy as np
from scipy.signal import spectrogram, chirp
import matplotlib.pyplot as plt

fs = 4000; T = 2
t = np.linspace(0, T, int(fs*T), endpoint=False)
# 線性 Chirp：50 Hz → 500 Hz
x = chirp(t, f0=50, f1=500, t1=T, method='linear')

f, t_spec, Sxx = spectrogram(x, fs=fs, nperseg=256, noverlap=192)
plt.pcolormesh(t_spec, f, 10*np.log10(Sxx+1e-12), shading='gouraud', cmap='viridis')
plt.ylabel('Frequency (Hz)'); plt.xlabel('Time (s)')
plt.title('Spectrogram of Linear Chirp')
plt.colorbar(label='Power (dB)'); plt.show()

7. 重要應用與定理

工程實務中的核心定理與常見陷阱

7.1 卷積定理 (Convolution Theorem) 與濾波器設計

$$y(t) = x(t) * h(t) \quad \longleftrightarrow \quad Y(j\omega) = X(j\omega) \cdot H(j\omega)$$

時域的卷積等於頻域的逐點相乘。這意味著：

  • 濾波 = 頻域乘法：低通濾波器 $H(j\omega)$ 在高頻處為零，將高頻成分移除。
  • 快速卷積 (Fast Convolution)：利用 FFT 可以在 $O(N\log N)$ 時間完成卷積，比直接計算的 $O(N^2)$ 快得多。在影像處理和長 FIR 濾波器中廣泛使用。

7.2 Nyquist-Shannon 取樣定理 (Sampling Theorem)

取樣定理

$$f_s \geq 2 f_{\max}$$

取樣頻率 $f_s$ 必須至少是訊號最高頻率 $f_{\max}$ 的兩倍。$f_s/2$ 稱為 Nyquist 頻率。

若違反此條件，會產生混疊 (Aliasing)：高頻成分「偽裝」成低頻成分，且無法事後修正。

工程實務：實際系統中，取樣前必須加入抗混疊濾波器 (Anti-Aliasing Filter)（類比低通濾波器），且通常取樣率會選擇遠大於 2 倍（如 2.56 倍以上）以留有餘裕。

互動圖表：Aliasing 混疊現象

原始訊號頻率固定為 50 Hz。調整取樣頻率，觀察取樣後的訊號。當 $f_s < 2 \times 50 = 100$ Hz 時，會產生混疊。

7.3 功率譜密度 (Power Spectral Density, PSD)

PSD 描述訊號的功率如何在頻率上分佈，單位為 V²/Hz 或 W/Hz：

$$S_{xx}(f) = \lim_{T\to\infty} \frac{1}{T}\left|X_T(f)\right|^2$$

實務上常用 Welch 法（將訊號分段加窗做 FFT 後平均）來估計 PSD，這樣可以降低估計的變異數。

7.4 FFT 常見陷阱與注意事項

陷阱 1：忘記加窗

不加窗 = 矩形窗 → 嚴重的頻譜洩漏，可能完全淹沒微弱訊號。

陷阱 2：混淆 FFT bin 和實際頻率

第 $k$ 個 bin 對應的頻率是 $f_k = k \cdot f_s / N$。常見錯誤：忘記除以 $N$ 或搞混單位。

陷阱 3：未正確處理單邊/雙邊頻譜

FFT 輸出是雙邊頻譜。取單邊頻譜時，除了 DC 和 Nyquist 分量外，幅度需要乘以 2。

陷阱 4：以為零填充能提高「真正的」解析度

零填充只是頻域內插，不會增加新資訊。要真正提高頻率解析度，需要更長的觀測時間。

陷阱 5：取樣率不足（Aliasing）

沒有使用抗混疊濾波器就直接取樣 → 高頻訊號折疊到低頻區域，結果完全錯誤。

Python / MATLAB 對應程式碼

import numpy as np
from scipy.signal import welch
import matplotlib.pyplot as plt

# Welch 法估計 PSD
fs = 1000; N = 10000
t = np.arange(N) / fs
x = np.sin(2*np.pi*100*t) + 0.5*np.sin(2*np.pi*250*t) + np.random.randn(N)*0.3

f_psd, Pxx = welch(x, fs=fs, nperseg=1024, noverlap=512)
plt.semilogy(f_psd, Pxx)
plt.xlabel('Frequency (Hz)'); plt.ylabel('PSD (V²/Hz)')
plt.title('Power Spectral Density (Welch)'); plt.grid(); plt.show()
